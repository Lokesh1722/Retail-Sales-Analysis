Project Description

The Retail Sales Analysis project is a SQL-based data analysis system designed to analyze sales performance in a retail business. This project uses structured datasets to study customer behavior, product performance, and overall sales trends, helping businesses make data-driven decisions. 
The project focuses on real-world retail scenarios using relational databases and advanced SQL queries.  

Project Objectives

Analyze retail sales data using SQL   
Understand customer purchasing behavior  
Identify top-performing products              
Track sales trends and revenue   
Generate business insights for decision-making   

Technologies Used

Database: MySQL    
Language: SQL  
Tools: MySQL Workbench / phpMyAdmin           

Dataset Description

The project uses the following datasets:      
Customer – Customer details such as ID, name, and demographics       
Product – Product information including category and price              
Sales – Sales transactions including quantity, date, and revenue           

Key Features   
         
Sales performance analysis     
Customer-wise purchase analysis      
Product-wise revenue tracking  
Category-level sales insights           
Time-based sales trend analysis          

SQL Concepts Used

SELECT, INSERT, UPDATE    
PRIMARY KEY & FOREIGN KEY      
JOINS (INNER JOIN)      
GROUP BY & HAVING      
SUBQUERIES     
AGGREGATE FUNCTIONS (SUM, COUNT, AVG)        

Sample Analysis Queries

Total sales revenue by product category      
Top-selling products  
Customer purchase frequency    
Monthly and yearly sales trends    
Highest revenue-generating customers      

How to Run the Project

Install MySQL                
Open MySQL Workbench or phpMyAdmin       
Create a new database   
Import the provided .sql file      
Load the CSV files if required
Execute SQL queries to analyze data
